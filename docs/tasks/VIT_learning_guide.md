## Part 1：背景知识速通

### 1.1 什么是图像分割？跟分类、检测有什么区别？

**一句话**：分类是"这张图里有猫"，检测是"猫在左上角那个框里"，分割是"这些像素属于猫，那些像素属于背景"。

| 任务 | 输出粒度 | 例子 |
|---|---|---|
| 分类 | 一张图一个标签 | "这是脑部 MRI" |
| 检测 | 一个/多个边界框 | "肿瘤在这个矩形区域" |
| **语义分割** | **每个像素一个标签** | **"这个像素是肿瘤，那个是正常组织"** |

在医学影像中，我们需要精确到像素级别知道肿瘤的边界，所以用分割而不是检测。

### 1.2 U-Net 的核心思路

U-Net 就像一个"压缩再解压"的过程：

```
原图 256x256
   ↓ 编码器（下采样，压缩）
   ↓ 每一层提取更抽象的特征，尺寸减半
bottleneck 16x16（最浓缩的特征）
   ↓ 解码器（上采样，恢复）
   ↓ 每一层恢复空间分辨率，尺寸翻倍
输出 256x256 的 mask
```

**关键设计 — Skip Connection（跳跃连接）**：
- 编码器压缩的时候丢失了很多细节（边缘、纹理）
- 跳跃连接把编码器每一层的细节"直接搬运"到解码器对应的层
- 类比：你先把一张照片缩小（丢细节），然后放大回来（模糊了），但如果你保留了原图的边缘信息直接叠上去，放大后的图就清晰多了

### 1.3 什么是 Vision Transformer？它跟 CNN 的本质区别

**CNN（卷积神经网络）**：用一个小滑动窗口（比如 3x3）在图上滑动，每次只看"局部"信息。就像用放大镜看图片——看得很细但视野有限。

**Vision Transformer（ViT）**：把图片切成小块（patches），让每个小块都能"看到"图片里所有其他小块。就像站远了看整张图——能看到全局关系，但可能忽略细节。

| 特性 | CNN | ViT |
|---|---|---|
| 感受野 | 局部（需要很多层才能看到全局） | **全局**（一层就能看到所有位置） |
| 对小细节 | 强 | 弱（patch 边界可能丢信息） |
| 对全局关系 | 弱（需要堆叠很多层） | 强（自注意力天然全局） |
| 计算量 | 相对低 | 较高（随图像尺寸平方增长） |

### 1.4 Patch Embedding：把图片变成"句子"

Transformer 最初是为文字设计的——把一个句子拆成一个个单词，再处理。

Vision Transformer 怎么处理图片？**把图片切成小方块，每个方块当作一个"单词"**。

```
原图 256x256，patch_size=16
→ 切成 (256/16)² = 16x16 = 256 个 patch
→ 每个 patch 是 16x16x3 的小图块
→ 用一个线性层（实际上是个卷积）把每个 patch 变成一个向量
→ 得到 256 个向量，每个维度是 embed_dim（比如 256 维）
→ 就像一个有 256 个"单词"的"句子"
```

### 1.5 自注意力机制的直觉

自注意力做的事情：**让每个 patch 去"询问"所有其他 patch："你跟我有关系吗？有多少关系？"**

具体步骤：
1. 每个 patch 生成三个向量：Q（Query，"我在找什么"）、K（Key，"我是什么"）、V（Value，"我的内容"）
2. 用 Q 和所有 K 做点积 → 得到"关注度分数"
3. 用 softmax 归一化 → 变成权重
4. 用权重对所有 V 加权求和 → 得到融合了全局信息的新表示

**类比**：你在课堂上听讲（你是一个 patch），你会不自觉地更关注跟你相关的同学的发言（注意力权重高），而忽略不相关的发言（权重低）。最终你获得的信息是所有人发言的加权总结。

### 1.6 位置编码：告诉 Transformer "你在哪"

Transformer 本身**不知道顺序**——它看到的只是 256 个向量，不知道哪个在左上角、哪个在右下角。

位置编码就是给每个 patch 加一个"坐标标签"：
- "我是第 (3, 5) 位置的 patch"
- 本项目用的是**可学习的位置编码**（`nn.Parameter`），让网络自己学出最优的空间关系表示

为什么重要？没有位置编码的话，把图片里所有 patch 打乱顺序，模型的输出完全一样——这显然不合理。

## Part 2：本项目详解

### 2.1 TransUNet 为什么把 ViT 和 CNN 结合？

一句话：**ViT 看全局但丢细节，CNN 看细节但缺全局，组合起来互补。**

TransUNet 的思路：
1. 用 CNN 编码器提取多尺度局部特征（保留边缘、纹理等细节）
2. 用 ViT 编码器提取全局特征（理解肿瘤和周围组织的空间关系）
3. 把两者在 bottleneck 层融合
4. 用 U-Net 风格的解码器 + 跳跃连接恢复到原始分辨率

这就像一个团队：CNN 是"细节控"，ViT 是"大局观"，最后合作给出最终答案。

### 2.2 完整数据流（Tensor Shape 变化）

以 TransUNet-Lite 为例，输入 `(B, 3, 256, 256)`：

```
输入图像: (B, 3, 256, 256)
           |
     +-----+------+
     |             |
  ViT Encoder    CNN Encoder
     |             |
  PatchEmbed     enc1: (B, 32, 256, 256)  ← skip1
  (B,256,256)    pool → enc2: (B, 64, 128, 128)  ← skip2
     ↓           pool → enc3: (B, 128, 64, 64)   ← skip3
  6x Transformer pool → enc4: (B, 256, 32, 32)   ← skip4
     ↓           pool → out: (B, 256, 16, 16)
  reshape          |
  (B,256,16,16)    |
     |             |
     +-----+------+
           |
     Concatenate: (B, 256+256, 16, 16) = (B, 512, 16, 16)
           |
     1x1 Conv Fusion: (B, 512, 16, 16)
           |
     UpBlock1 + skip4: (B, 256, 32, 32)
     UpBlock2 + skip3: (B, 128, 64, 64)
     UpBlock3 + skip2: (B, 64, 128, 128)
     UpBlock4 + skip1: (B, 32, 256, 256)
           |
     1x1 Conv: (B, 1, 256, 256)  ← 输出 mask
```

### 2.3 损失函数为什么用 Dice + BCE？

**问题**：脑部 MRI 中肿瘤区域很小（可能只占图像的 1-5%），背景占 95%+。

**单独用 BCE（二元交叉熵）的问题**：
- 模型可以通过"全部预测为背景"就获得 95% 准确率
- BCE 对每个像素平等计算损失，小目标贡献的梯度被淹没

**单独用 Dice Loss 的问题**：
- Dice 直接优化 F1 分数，天然关注前景区域
- 但梯度不如 BCE 稳定，训练初期可能震荡

**组合的好处**：
- Dice 负责"让模型关注小目标区域"
- BCE/Focal 负责"提供稳定的像素级梯度信号"
- `loss = 0.5 * Dice + 0.5 * BCE/Focal` → 两全其美

**Focal Loss 的额外好处**：对容易分类的像素（大量背景）降低权重，让模型更关注"难"的像素（肿瘤边界）。

### 2.4 Patient-Level Split 是什么？为什么不能按切片随机划分？

**问题场景**：一个患者有 30 张 MRI 切片，相邻切片长得几乎一样。

如果**按切片随机划分**：
- 同一个患者的第 15 张切片在训练集，第 16 张在验证集
- 模型"见过"几乎一样的图，验证集上的表现虚高
- 这叫**数据泄漏（data leakage）**

**Patient-Level Split 的做法**：
- 先把 110 个患者分成两组（77 个训练、33 个验证）
- 同一患者的所有切片只出现在一个组里
- 验证集里的患者在训练时完全没见过 → 结果更真实

本项目的实现：`src/dataset.py` 中的 `_split_indices_by_patient()` 函数，通过文件夹名识别患者 ID。

### 2.5 Dice=0.788 是什么水平？还能怎么提升？

**参考对比**：
- 二分类脑肿瘤分割，Dice 0.788 属于**合理的 baseline 水平**
- 原论文 TransUNet 在 Synapse 数据集上报告 ~0.78 Dice（多类别平均）
- 顶尖方法在 LGG 上可以达到 0.85+

**可能的提升方向**：
1. **预训练权重**：用 ImageNet 预训练的 ViT 权重初始化，而不是从头训练
2. **更多数据增强**：弹性变形（elastic deformation）、亮度/对比度增强
3. **更深的模型**：TransUNet-Full（768-dim, 12 layers）如果显存允许
4. **后处理**：CRF（条件随机场）或连通域分析去除小的假阳性
5. **更长训练**：100+ epochs，可能还没收敛完全

### 3.2 面试官最可能问的 10 个问题 + 参考答案

**Q1: 为什么选择 TransUNet 而不是纯 U-Net？**
> U-Net 的卷积核感受野有限，需要很多层才能捕获远距离依赖。TransUNet 用 ViT 一层就能看到全局，适合需要理解肿瘤和周围组织空间关系的医学影像任务。实验中我也保留了 U-Net 作为对比基线。

**Q2: 什么是 Patch Embedding？为什么不直接把整张图输入 Transformer？**
> 如果直接按像素做自注意力，256x256 的图就有 65536 个 token，计算量是 O(n²) ≈ 4.3 billion，完全不可行。Patch Embedding 把图切成 16x16 的块，只需要 256 个 token，计算量降到 O(256²) ≈ 65K，可以接受。

**Q3: 多头注意力比单头好在哪？**
> 多头让模型同时关注不同类型的关系。比如一个头可能学到关注纹理相似的区域，另一个头关注空间距离近的区域。就像团队合作，每个人负责不同维度的分析，最后综合起来比一个人的视角更全面。

**Q4: 为什么要做 patient-level split？**
> 同一个患者的相邻 MRI 切片几乎一样。如果随机划分，模型在验证集上"作弊"——它见过几乎相同的图。按患者划分后，验证集里的患者在训练时完全没见过，结果更可信。这是医学影像实验的基本要求。

**Q5: Dice Loss 跟交叉熵有什么区别？为什么组合用？**
> Dice 直接优化重叠度，对小目标（肿瘤只占 1-5% 像素）比 BCE 更敏感。但 Dice 的梯度不太稳定。BCE 提供稳定的像素级梯度。组合起来：Dice 负责关注小目标，BCE 负责稳定训练。

**Q6: Focal Loss 解决什么问题？**
> 标准 BCE 对所有像素平等对待，但 95% 是"容易"的背景像素。Focal Loss 给容易分类的样本降低权重（乘以 (1-p)^γ），让模型把更多注意力放在难以分类的肿瘤边界像素上。

**Q7: 0.788 的 Dice 够好吗？还能怎么提升？**
> 作为从头训练的 baseline 来说合理。主要提升方向有三个：加载 ImageNet 预训练的 ViT 权重、增加弹性变形等数据增强、延长训练到 100+ epochs。如果用预训练权重，预计能提到 0.83-0.85。

**Q8: 你的 ViT 用了预训练权重吗？**
> 目前没有，是从随机初始化训练的。这也是 Dice 没有更高的主要原因之一。ViT 的参数多，从头训练在小数据集上容易欠拟合。如果时间允许，下一步就是加载 ImageNet 预训练权重。

**Q9: 这个项目和你的 YOLO 项目有什么不同？**
> YOLO 是工业检测项目，侧重工程部署能力（TensorRT 加速、实时推理）；TransUNet 是医学分割项目，侧重算法深度理解（Transformer 机制、损失函数设计、数据泄漏防护）。两个项目一个展示工程落地，一个展示研究理解，形成互补。

**Q10: 如果让你做 BraTS 四分类，你会怎么改？**
> 主要改三个地方：数据加载改成 NIfTI 格式的多模态输入（T1、T2、FLAIR、T1ce 四个通道）；输出通道从 1 改成 4，损失换成 CE+Dice；评估指标按每个肿瘤区域分别计算 Dice。代码里已经预留了多分类的 dataset 和 loss，但还没在真实数据上跑通。
