## 一、项目定位

这是简历上的核心项目，目标岗位是外企 CV 方向（博世、西门子、Cognex 等）。

面试时这个项目承载两个任务：

1. 证明你有完整的「数据 → 训练 → 评估 → 部署 → 服务化」项目经验
2. 证明你能用业界主流工具（YOLO + ONNX + FastAPI + Docker）解决工业场景的实际问题

时间紧迫，追求快速出成果，不追求理论完美。先跑通 baseline，再逐步打磨。

## 二、项目目标

|项目|内容|
|---|---|
|数据集|NEU-DET 钢材表面缺陷数据集|
|缺陷类别|6 类：Crazing、Inclusion、Patches、Pitted Surface、Rolled-in Scale、Scratches|
|样本量|约 1800 张（每类约 300 张）|
|模型|YOLOv8n（nano 版本，轻量、边缘部署友好）|
|训练指标|mAP@0.5 > 0.70|
|部署验证|ONNX 导出 + Python ONNX Runtime 推理，精度对齐|
|服务化|FastAPI + Docker（简历里作为项目1的子项）|
|SAM 集成|YOLO 检测框 → SAM 精细分割（简历亮点）|
|最终交付|GitHub 精品项目（README + Demo GIF + 清晰结构）|

**设计说明：**

- `scripts/` 放一次性脚本（数据准备、训练、评估）

- `api/` 放服务化相关代码（Week 2 再建）
- `configs/` 用 yaml 管理超参，调参时不用改代码，只改配置

## 五、分步计划（项目蓝图）

### Step 1：数据准备

1. 下载 NEU-DET 数据集
2. 分析数据分布：每类样本数、图片尺寸、标注格式
3. 转换为 YOLO 格式（NEU-DET 原始是 VOC XML）
4. 划分 train/val/test（8:1.5:0.5 或 8:2）
5. 创建 `data.yaml`

**产出：** `data/` 结构完整，`data.yaml` 可用，数据分析截图保存到 `docs/assets/`

### Step 2：基线训练

```bash
yolo detect train model=yolov8n.pt data=data.yaml epochs=50 imgsz=640 batch=16
```

**产出：** baseline mAP@0.5 数值（预期 0.50-0.65），loss 曲线

### Step 3：调参优化

|实验|变量|说明|
|---|---|---|
|实验 A|imgsz|512 vs 640 vs 800|
|实验 B|学习率|lr0=0.01 vs 0.001|
|实验 C|增强策略|默认 vs 调整 mosaic/mixup|
|实验 D|模型大小|YOLOv8n vs YOLOv8s|

每组实验只改一个变量。用 `configs/train_config.yaml` 管理超参。

**产出：** mAP@0.5 > 0.70 的最优配置，完整对比表格

### Step 4：结果分析

1. PR 曲线（ultralytics 自动生成）
2. 混淆矩阵：哪些缺陷容易互相误判
3. 误检案例分析：3-5 个典型 case
4. 按类别 AP：哪类最难检测，为什么

**产出：** `docs/experiment_log.md` 完整记录

### Step 5：ONNX 导出 + 推理验证

```bash
yolo export model=runs/detect/train/weights/best.pt format=onnx simplify=True
```

1. 编写 `src/detector.py`（核心推理类）和 `scripts/inference_onnx.py`
2. 对比 PyTorch vs ONNX 精度（mAP 差异应 < 0.01）
3. 记录 FPS 对比（CPU + GPU）

**产出：** `best.onnx` + 精度对齐报告 + 性能对比表

### Step 6：SAM 集成

YOLO 检测 bbox → 取中心点作为 SAM prompt → 生成精细 mask

**产出：** 5 张 YOLO框 vs SAM精细mask 对比图

### Step 7：GitHub 美化

1. 英文 README（项目亮点 → Demo GIF → 快速开始 → 实验结果 → 技术栈）
2. 录制 60 秒 Demo GIF
3. 确保 3 行命令能跑通推理
4. requirements.txt + LICENSE

### Step 8：FastAPI + Docker（Week 2）

1. `POST /detect`：上传图片 → ONNX 推理 → 返回 JSON
2. `GET /health`：健康检查
3. Dockerfile 容器化
4. 简单压测记录 QPS

**产出：** Docker 镜像 + API 能跑通 + 性能数据
