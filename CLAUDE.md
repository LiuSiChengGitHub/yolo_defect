# CLAUDE.md

Project context for AI assistants working on this codebase.

## Project Overview

**yolo_defect** — Industrial steel surface defect detection using YOLOv8, targeting mAP@0.5 > 0.70. Resume/portfolio project with emphasis on code quality and reproducibility.

- **Dataset:** NEU-DET (1,800 images, 6 classes, 200x200px, ~28MB, committed to git)
- **Model:** YOLOv8n (Ultralytics)
- **Goal:** Train → Evaluate → Export ONNX → FastAPI service
- **Conda env:** `yolo_defect` (Python 3.9, PyTorch 2.0, CUDA 11.8)

## Key Conventions

- **All paths must be relative** — The user works on two machines with different root paths. Never hardcode `D:\` or any absolute path in code.
- **Scripts use `os.path.dirname(os.path.abspath(__file__))` to locate project root** — This is the standard pattern across all scripts.
- **Code comments in English** — User prefers English comments, Chinese only in README and docs.
- **argparse for all scripts** — No hardcoded paths, always provide CLI arguments with sensible defaults.
- **`if __name__ == "__main__":`** — Required for all scripts.

## Directory Layout

```
scripts/     → One-off CLI scripts (data prep, train, eval, export, inference)
src/         → Reusable modules (detector.py used by both inference script and FastAPI)
configs/     → YAML hyperparameter configs (not CLI args, for experiment traceability)
api/         → FastAPI service (Week 2, currently empty)
data/NEU-DET/→ Original dataset (DO NOT modify)
data/        → YOLO-format data generated by prepare_data.py
models/      → Exported ONNX models (gitignored)
runs/        → Training outputs (gitignored)
docs/tasks/  → Project task prompts (DO NOT modify)
docs/assets/ → Plots, PR curves, demo GIFs
```

## Dataset Details

- 6 classes: crazing, inclusion, patches, pitted_surface, rolled-in_scale, scratches
- Class mapping: crazing=0, inclusion=1, patches=2, pitted_surface=3, rolled-in_scale=4, scratches=5
- `rolled-in_scale` has a hyphen — use known class name prefix matching, not underscore splitting
- Images stored in subdirectories by class: `images/{class_name}/{class_name}_N.jpg`
- Annotations flat: `annotations/{class_name}_N.xml` (VOC XML format)
- Pre-split: train (~240/class), validation (~60/class) — do NOT re-split

## Common Commands

```bash
conda activate yolo_defect
python scripts/prepare_data.py          # VOC XML → YOLO TXT
python scripts/data_analysis.py         # Dataset stats & plots
python scripts/train.py                 # Train (reads configs/train_config.yaml)
python scripts/evaluate.py --weights runs/detect/train/weights/best.pt
python scripts/export_onnx.py --weights runs/detect/train/weights/best.pt
python scripts/inference_onnx.py --model models/best.onnx --image test.jpg
```

## Do NOT

- Modify files under `data/NEU-DET/` — this is the original dataset
- Modify files under `docs/tasks/` — these are project prompts
- Modify `.gitignore` or `environment.yml` unless explicitly asked
- Use absolute paths anywhere in code
- Add model weights (.pt, .onnx) to git — they are gitignored
